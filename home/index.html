About IN-CORE

The National Institute of Standards and Technology (NIST) funded the multi-university five-year Center of Excellence for Risk-Based Community Resilience Planning (CoE), headquartered at Colorado State University, to develop the measurement science to support community resilience assessment. Measurement science is implemented on a platform called Interdependent Networked Community Resilience Modeling Environment (IN-CORE). On IN-CORE, users can run scientific analyses that model the impact of natural hazards and resiliency against the impact on communities. The IN-CORE platform is built on a Kubernetes cluster with Docker container technology. On the cluster, 1) IN-CORE Lab, customized JupyterLab, 2) pyIncore, Python library of scientific analyses, 3) IN-CORE Web Service API, and 4) IN-CORE Web Tools, light-weight web applications are implemented.

pyIncore - Python Library

pyIncore is a Python package to analyze and visualize various hazard (earthquake, tornado, hurricane etc.) scenarios developed by the Center for Risk-Based Community Resilience Planning team from NCSA. The development is part of NIST sponsored IN-CORE (Interdependent Networked Community Resilience Modeling Environment) initiative. pyIncore allows users to apply hazards on infrastructure in selected areas. Python framework acceses underlying data through local or remote services and facilitates moving and synthesizing results.

IN-CORE Web Service API

IN-CORE uses a service oriented architecture with a REST API for communicating with the different services. IN-CORE currently maintains 4 different services: The Authentication Service supports secure LDAP authentication, and returns tokens to approved users to access the other services; Data Service provides basic capabililities to fetch/store data from file storage; Fragility service that supports fragilities and fragility mapping. Given a set of attributes, the service can find a matching fragility and return it to the user; The Hazard Service supports creating model based or data based hazards, including earthquakes, tsunamis, tornadoes, and hurricanes.

IN-CORE Lab

The IN-CORE Lab on JupyterHub provides a web-based working environment for multiple scientific research users. JupyterHub is a multi-user Hub, which can spawns, manages, and proxies multiple instances of the single-user JupyterLab. IN-CORE Lab, customized JupyterLab, enables user to work with documents and writing code, using Jupyter notebooks, text editors, terminals, and custom components in a flexible, integrated, and extensible manner. Python is enabled, and Pyincore library together with a few popular scientific libraries are preinstalled. Custom components such as IN-CORE login plugin, IN-CORE Web Tools plugin improves the code writing experience of the user. Code consoles and Kernel-backed documents provides transient scratchpads for running interactively with full support of rich output.

IN-CORE Web Tools

The web front-end provides a user interface for interacting with the service layer. It provides a username/password login interface that communicates to the authentication services. It enables user to browse and search the datasets, hazards and fragilities, view the metadata and visualizations, and download the datasets.
